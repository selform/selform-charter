# Decision: Avoid the “user” framing by default
- Decision: In this repository, do not default to calling participants “users.” Prefer *participant / practitioner / subject*.
- Context: “User” implies an external consumption relationship and pulls discussion back into product logic.
- Rationale: Selform’s frame is about subject-formation and ethical human–technology relations; vocabulary should preserve that frame.
- Impact: Templates and docs should use non-“user” language except when explicitly discussing why “user” is misleading.
