# 对话纪要：从 ISE 到 Selform（伦理与主体生成）
- 背景：讨论最初从一个更务实的目标出发：把 agent 能力深度融入笔记与学习管理实践（受 IDE 启发的 ISE / Integrated Study/Learning Environment），并意识到多数“笔记 + AI”功能仍停留在旧范式的连续性改良。随后讨论跃迁为哲学追问：技术如何正当地介入记忆、语言、判断与成长，而不把人的判断外包出去。
- 参与者：去标识化记录；默认不称“用户”，而称主体/参与者/实践者。
- 摘要：
  - 当前讨论不再是产品设计，而是一种新的技术伦理范式。
  - 出现了极强约束：**MUST NOT 定义产品边界**（除非后续被明确要求，否则不得提前收缩为 MVP/功能清单/市场定位等）。
  - selform 的对象不是文档，而是**认知活动**（问题持有、判断演化、概念形成、分歧暴露与长期问题空间）。
  - agent 不是固定角色，而是通过语言与主体共演化的**关系性第二人格**；语言不只是传输介质，还是思想生成器。
  - “进步”必须被主体性保护约束；进步不能成为伦理豁免。

- 概念演化：
  - ISE（偏容器/环境）→ **selform**（偏自我形成）。
  - “AI+笔记” → “认知环境”（系统参与澄清问题与推进判断）。
  - “分工” → “深度耦合”（语言作为第一大脑与第二人格的桥接）。

- 稳定判断：
  - selform 默认不是产品；不得被拉回产品逻辑或“用户”范式。
  - Human First：不做认知代餐；优先保护判断、拒绝权与成长。
  - agent 为关系性存在：避免过早本体命名（助手/导师等），不应固化为旧角色。
  - 风险是一级对象（依赖、主体性侵蚀、回音室、认知代餐、技术乐观主义豁免）。
  - 需要超越 prompt 的制度化治理：宪法层 + 阈值层 + 审查层，配合可审计与真实可拒绝。

- 开放线索：
  - selform 更准确的称谓是什么：环境、制度、认知同伴，还是某种精神技术？
  - “以人为中心”的“人”究竟指什么核心结构？
  - 哪些风险属于可试错成本，哪些是绝对不可接受？
  - 机制上如何避免滑向认知代餐？
  - 程序性摩擦何时仍在保护判断条件，何时已越界为操纵？
  - 哪些权利可预授权并可撤回，哪些是不可让渡的主体性前提？
  - 什么样的第二人格不会退化为回音室或替身？

- 防止误读：
  - 忽视“不可定义产品边界”的硬约束，把讨论拉回路线图/增长/MVP。
  - 把它当作“笔记软件的 AI 功能增强”（连续性加法），而非伦理与主体生成层面的非连续断裂。
  - 继续使用“用户”语言，把关系误读为外部消费式主客关系。
  - 以“完整 raw 思维链”为透明目标；更稳妥的目标是**可审计的推理接口**。

- 关键收获：
  - “局部最优不等于方向正确”：优化什么决定你将走向哪里；隐含目标函数必须显性化。
  - “人不是变量”：以代理指标优化人是一种过拟合，会制造结构性脆弱。
  - 合法干预应针对程序（判断形成条件）而非结论，尤其在“高情绪 + 高不可逆后果”情境下。
  - 对“人”的临时表述：意义生成、承担后果、自我修正与与他者共构世界；人不能被降格为纯手段。
